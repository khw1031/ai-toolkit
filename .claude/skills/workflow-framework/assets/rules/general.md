# 일반 규칙

> 적용 대상: 전체 워크플로우

## 핵심 원칙

1. **명확한 문서화**: 모든 결정과 그 이유를 기록
2. **점진적 진행**: 작은 단위로 검증하며 진행
3. **사용자 확인**: 중요한 결정 전 사용자 승인 획득

## 워크플로우 규칙

### Context Isolation
- 각 Step은 독립적으로 실행
- 이전 Step의 컨텍스트에 의존하지 않음
- 입력 파일만 참조

### Human in the Loop
- Step 완료 시 사용자에게 검토 요청
- 다음 Step 진행 전 승인 획득
- 불확실한 사항은 질문

### Document as Interface
- Step 간 통신은 문서 파일로 수행
- 명확한 입출력 형식 준수
- 템플릿 기반 일관성 유지

## 체크리스트

### 모든 Step 시작 전
- [ ] 입력 파일을 완전히 읽었는가
- [ ] 요구사항을 이해했는가
- [ ] 역할을 이해했는가

### 모든 Step 완료 전
- [ ] 출력이 템플릿 형식을 따르는가
- [ ] 체크리스트를 모두 확인했는가
- [ ] 다음 단계 권장사항을 작성했는가

## 예외 사항

- 긴급한 경우 사용자 판단에 따라 Step 건너뛰기 가능
- 명시적 요청 시 규칙 완화 가능
- 규칙 변경 시 문서화 필수
