# Progressive Disclosure 원칙

> 정보를 필요한 시점에 필요한 만큼만 로드하여 컨텍스트 윈도우를 효율적으로 사용합니다.

## 목적

LLM의 컨텍스트 윈도우는 제한된 자원입니다. 모든 정보를 한 번에 로드하면 핵심 내용이 희석되고 성능이 저하됩니다. Progressive Disclosure는 정보를 3단계로 나누어 **필요할 때만 로드**함으로써 컨텍스트 효율성을 극대화합니다.

## 3단계 로드 모델

| 단계 | 로드 시점 | 토큰 | 내용 |
|------|----------|------|------|
| 1단계 | 항상 | ~100 | 이름, 설명, 트리거 키워드 |
| 2단계 | 활성화 시 | <5000 | 핵심 규칙, 필수 지침 |
| 3단계 | 요청 시 | 무제한 | 예제, 상세 문서, 스크립트 |

## 적용 대상

- **Skills** - SKILL.md 작성 시
- **Rules** - 규칙 파일 작성 시  
- **Agents** - 서브에이전트 정의 시
- **Prompts** - 시스템 프롬프트 구조화 시

## 파일 구조

```
progressive-disclosure/
├── RULE.md          # 핵심 원칙 (2단계)
└── references/      # 상세 가이드 (3단계)
    ├── skills.md
    ├── rules.md
    ├── agents.md
    ├── commands.md
    └── prompts.md
```

## 핵심 체크리스트

- [ ] 1단계 정보만으로 "무엇인지" 파악 가능한가?
- [ ] 2단계 내용이 5000 토큰 이하인가?
- [ ] 상세 내용이 별도 파일로 분리되어 있는가?
